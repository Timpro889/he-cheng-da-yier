给Claude Code的任务描述（直接复制使用）

用 React + Vite + TypeScript + Tailwind CSS 做一个"合成大一二"微信小游戏。

## 核心概念
物理碰撞合成游戏：玩家控制一二/布布角色下落，相同角色碰撞合成更大版本，最终目标是合成"一二布布合体"。

## 已有素材
- 一二和布布的角色图片（各种尺寸：迷你/小号/中号/大号/超大）
- 每个角色有5个表情状态

## 核心需求

### 1. 游戏主界面
- 顶部：当前分数 + 最高纪录
- 中间：游戏区域（物理世界）
- 底部：角色预览（下一个要投放的角色）
- 左右滑动控制投放位置，点击投放

### 2. 物理引擎
- 使用 Matter.js 或 Planck.js
- 角色有圆形碰撞体（自然滚动）
- 重力下落，碰撞反弹
- 边界检测（左右墙壁+底部）

### 3. 合成系统
| 等级 | 一二 | 布布 |
|:---|:---|:---|
| Lv.1 | 迷你一二(委屈🥺) | 迷你布布(歪头🤔) |
| Lv.2 | 小号一二(眨眼😉) | 小号布布(憨笑😅) |
| Lv.3 | 中号一二(叉腰😏) | 中号布布(张开双臂🤗) |
| Lv.4 | 大号一二(欢呼🎉) | 大号布布(竖拇指👍) |
| Lv.5 | 超大一一二(捧脸🥰) | 超大布布(wink😘) |

合成规则：
- 🐼+🐼=大🐼（一二合成一二）
- 🐻+🐻=大🐻（布布合成布布）
- 🐼+🐻=❤️"贴贴"状态（特殊效果：下次合成双倍成长）

### 4. 合成动画（关键！）
合成触发时：
1. 两个角色碰撞弹开（物理感）
2. 爆出爱心❤️ + 星星✨粒子
3. 新角色"duang"弹性出现
4. 新角色做专属表情动作（2秒）
5. 文字气泡："一二又变胖啦~" / "布布来保护你"

### 5. 胜利条件
合成"一二布布合体"（终极形态）：
- 一二骑在布布肩上
- 经典"宠妻"pose
- 全屏庆祝动画
- 播放语音："一二最可爱了" + "布布最好了"

### 6. 失败条件
角色堆叠超出顶部红线
- 游戏结束画面
- 显示"一二委屈了，布布快来安慰"
- 分享按钮："我的最高纪录是XXX"

### 7. 响应式
- 手机端：竖屏，全屏触摸
- 桌面端：居中显示，支持鼠标操作
- 游戏区域比例：9:16（适配手机屏幕）

## 技术栈
React 18 + TypeScript + Vite + Tailwind CSS + Matter.js（物理引擎）+ Framer Motion（动画）+ Zustand（状态管理）

## 交付
- 完整可运行的项目代码
- 角色配置文件（方便替换素材）
- 物理参数配置文件（重力、弹性、摩擦力）
- 构建后可部署到微信小游戏/网页

## 加分项

- 皮肤系统（不同场景主题）
- 音效系统（合成语音、背景音乐）

一个关键建议

 先做"静态版本"验证核心体验，再加物理引擎。

Claude Code可以先做一个"点击合成"的静态版本（两个角色放上去，点击就合成），您玩着觉得对味了，再加上物理碰撞。这样避免物理参数调半天，最后发现玩法不对。


更好的方案：保持角色比例 + 圆形碰撞体

视觉呈现          物理碰撞
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ╭────╮        
  │ 🐼 │    ←   角色保持原有比例（PNG透明底）
  ╰────╯        
    ◯          ←   底部圆形碰撞体（隐形）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━


关键设计：
- 视觉层：角色图片保持原有宽高比（比如 1:1.2），不强制正方形
- 物理层：底部放一个圆形碰撞体，角色在上面自然滚动
- 碰撞检测：用圆形计算，视觉和物理分离

• 影响: 碰撞反弹

**摩擦力**

• 参数: 摩擦力

• 影响: 滚动距离

• 调优难度: ⭐️⭐️⭐️⭐️ 影响堆叠策略

**角色尺寸差**

• 参数: 角色尺寸差

• 影响: 合成节奏

• 调优难度: ⭐️⭐️⭐️⭐️⭐️ 最难调，决定游戏时长

2. 合成策略深度

简单版：随机投放，碰运气

更新后的任务描述（关键修改）

### 3. 角色设计
- 视觉：保持一二/布布原有比例（约1:1.2），PNG透明底
- 物理：底部圆形碰撞体（半径=图片宽度的40%）
- 碰撞体隐形，只用于物理计算

### 4. 合成系统（新增策略维度）
- 相同角色合成：🐼+🐼=大🐼
- 跨角色"贴贴"：🐼+🐻=❤️（下次合成双倍成长）
- 连锁合成：一次碰撞触发多次合成，分数翻倍
